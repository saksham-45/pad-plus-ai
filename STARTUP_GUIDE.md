# 🚀 Руководство по запуску PAD+ AI

## Исправленные проблемы

### 1. WebSocket подключение к X-Ray
**Проблема**: WebSocket connection to 'ws://localhost:8080/api/v1/xray/ws' failed

**Причина**: 
- Vite proxy не был настроен для правильного перенаправления WebSocket запросов
- Frontend пытался подключиться напрямую к порту 8080, минуя прокси

**Решение**: 
1. Добавлен явный прокси для XRay WebSocket в `frontend/vite.config.js`
2. Изменен URL в `frontend/src/pages/XRayPage.jsx` для использования порта Vite (5174):
```javascript
// Было (прямое подключение к backend):
const XRAY_WS_URL = `ws://${window.location.hostname}:8080/api/v1/xray/ws`;

// Стало (через Vite proxy):
const XRAY_WS_URL = `ws://${window.location.hostname}:5174/api/v1/xray/ws`;
```

### 2. 401 Unauthorized при загрузке ключей
**Проблема**: GET http://localhost:5174/api/v1/keys?offset=0&limit=100 401 (Unauthorized)

**Причина**: 
- Если пользователь не аутентифицирован или токен истек, API возвращает 401
- Frontend пытался загрузить ключи даже при отсутствии валидного токена

**Решение**: 
- Добавлена проверка наличия токена перед запросом
- При получении 401 ответа выполняется автоматический выход из системы
- Улучшена обработка ошибок в `frontend/src/App.jsx`

### 3. Улучшена обработка ошибок WebSocket
**Изменения в `frontend/src/pages/XRayPage.jsx`**:
- Добавлена очистка таймаутов перед новым подключением
- Улучшено логирование для отладки
- Автоматическое переподключение только при некорректном закрытии соединения

## 📋 Инструкция по запуску

### Шаг 1: Подготовка окружения

1. Убедитесь, что у вас установлен Python 3.9+ и Node.js 16+
2. Активируйте виртуальное окружение:
   ```bash
   # Windows (cmd)
   venv\Scripts\activate
   
   # Windows (PowerShell)
   .\venv\Scripts\Activate.ps1
   ```

### Шаг 2: Проверка .env файла

Убедитесь, что файл `.env` содержит необходимые настройки:
- `SUPABASE_URL` и `SUPABASE_KEY` - для подключения к базе данных
- `ENCRYPTION_KEY` - для шифрования API ключей

### Шаг 3: Запуск системы

**Вариант A: Использование start.bat (рекомендуется)**

Просто запустите `start.bat` в корне проекта:
```cmd
start.bat
```

Скрипт автоматически:
1. Освободит порты 8080 и 5174 если они заняты
2. Запустит backend на порту 8080
3. Запустит frontend на порту 5174
4. Откроет браузер

**Вариант B: Ручной запуск**

1. Запустите backend:
   ```bash
   cd backend
   uvicorn main:app --reload --port 8080
   ```

2. В новом терминале запустите frontend:
   ```bash
   cd frontend
   npm run dev
   ```

3. Откройте браузер: http://localhost:5174

### Шаг 4: Проверка работы

1. **Проверка backend**:
   - Откройте http://localhost:8080/health
   - Должен вернуться статус "healthy"

2. **Проверка frontend**:
   - Откройте http://localhost:5174
   - Зарегистрируйтесь или войдите в систему
   - Перейдите на вкладку "X-Ray" - WebSocket должен подключиться

### Шаг 5: Остановка системы

Используйте `stop.bat` или закройте терминалы с backend и frontend.

## 🔧 Устранение неполадок

### Порты заняты

Если порты 8080 или 5174 заняты:

**Windows (PowerShell)**:
```powershell
# Найти процесс на порту 8080
netstat -ano | findstr :8080

# Убить процесс (замените PID на нужный)
taskkill /F /PID <PID>
```

### Backend не запускается

1. Проверьте наличие всех зависимостей:
   ```bash
   pip install -r requirements.txt
   ```

2. Проверьте .env файл - все ли переменные заданы

### Frontend не запускается

1. Установите зависимости:
   ```bash
   cd frontend
   npm install
   ```

2. Очистите кэш:
   ```bash
   rm -rf node_modules
   npm install
   ```

### WebSocket не подключается

1. Проверьте, что backend запущен на порту 8080
2. Проверьте консоль браузера на наличие ошибок
3. Попробуйте перезагрузить страницу

### 401 Unauthorized

1. Убедитесь, что вы вошли в систему
2. Проверьте токен в localStorage
3. Попробуйте выйти и войти заново

## 📝 Примечания

- Frontend использует Vite proxy для перенаправления API запросов на backend
- WebSocket подключения работают через тот же прокси
- При разработке изменения в коде frontend применяются автоматически (HMR)
- Backend работает в режиме reload - изменения применяются автоматически
