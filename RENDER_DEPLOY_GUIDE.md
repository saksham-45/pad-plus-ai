## 📋 Шаги для деплоя на Render

### Шаг 1: Подключите GitHub к Render
1. Зайдите в [Render Dashboard](https://dashboard.render.com/)
2. Нажмите **"New +"** → **"Blueprint"**
3. Подключите ваш GitHub аккаунт
4. Выберите репозиторий `Ovladimirovich/PAD-AI-v3.5`

### Шаг 2: Настройте Blueprint
1. Render автоматически обнаружит `render.yaml`
2. Убедитесь, что сервисы созданы:
   - **padplus-ai-backend** (Python Web Service)
   - **padplus-ai-frontend** (Static Site)

### Шаг 3: Настройте переменные окружения
Для backend сервиса установите:
- `OPENROUTER_API_KEY` - ваш ключ (если хотите предоставить системный)
- `DATABASE_URL` - уже настроена на Supabase
- Остальные переменные уже в `render.yaml`

### Шаг 4: Запустите деплой
1. Нажмите **"Create Blueprint"**
2. Render автоматически соберёт и задеплоит оба сервиса
3. Ожидайте завершения сборки (5-10 минут)

## 🔧 Проверка деплоя

### 1. Backend проверка
- Откройте `https://padplus-ai-backend.onrender.com/health`
- Должен вернуть `{"status": "ok"}`

### 2. Frontend проверка  
- Откройте `https://padplus-ai-frontend.onrender.com`
- Интерфейс должен загрузиться

### 3. Тестирование чата
1. Откройте настройки (⚙️)
2. Добавьте ваш API ключ для OpenRouter
3. Отправьте сообщение в чат
4. Проверьте, что ответ приходит

## 🎯 Что работает после деплоя

### ✅ Функциональность:
- **Чат без API ключей** - через fallback систему
- **Пользовательские провайдеры** - каждый может настроить свои ключи
- **Изоляция пользователей** - настройки не пересекаются
- **Автоматическая сессия** - генерируется при первом посещении

### 🔒 Безопасность:
- API ключи хранятся только в сессии (24 часа)
- Нет сохранения ключей в базе данных
- Автоматическая очистка истекших сессий

## 🚨 Возможные проблемы и решения

### Проблема: Backend не запускается
**Решение:** Проверьте логи в Render Dashboard, убедитесь что все зависимости установлены

### Проблема: Frontend не может достучаться к backend
**Решение:** Убедитесь что `VITE_API_URL` в frontend настроен правильный URL backend

### Проблема: Чат не работает с API ключами
**Решение:** Проверьте что API ключи валидные и провайдер включён в настройках

## 🔄 Обновления

### Для обновления кода:
1. Сделайте изменения в локальном репозитории
2. Закоммитьте: `git add . && git commit -m "Update"`
3. Отправьте: `git push origin main`
4. Render автоматически задеплоит изменения

### Для обновления переменных:
1. Зайдите в Render Dashboard
2. Выберите сервис → **Environment**
3. Измените переменные → **Save Changes**
4. Перезапустите сервис
