# Руководство по настройке провайдеров LLM

## Общие сведения

PAD+ AI поддерживает работу с несколькими LLM-провайдерами. Это руководство поможет вам настроить каждый из них через веб-интерфейс.

## Настройка провайдеров

### 1. OpenRouter (рекомендуемый)

#### Требуется:
- API-ключ от OpenRouter

#### Шаги:
1. Зайдите на [https://openrouter.ai/keys](https://openrouter.ai/keys)
2. Создайте новый API-ключ
3. Скопируйте ключ
4. В интерфейсе PAD+ AI:
   - Вставьте API-ключ в поле "API Key" для OpenRouter
   - Выберите желаемую модель из списка
   - Нажмите "Протестировать"
   - Если тест пройден успешно, провайдер будет активен

#### Популярные модели:
- Google Gemma 7B (бесплатно)
- Google Gemma 2 9B (бесплатно)
- Meta Llama 3 8B (бесплатно)
- Anthropic Claude 3 Sonnet
- OpenAI GPT-4

### 2. GigaChat (для российских пользователей)

#### Требуется:
- CLIENT_ID
- AUTHORIZATION_KEY (Base64-кодированный client_id:client_secret)

#### Шаги:
1. Зарегистрируйтесь на [https://developers.sber.ru/](https://developers.sber.ru/)
2. Получите доступ к GigaChat API
3. Создайте проект и получите credentials
4. Сформируйте AUTHORIZATION_KEY:
   ```bash
   echo -n "client_id:client_secret" | base64
   ```
5. В интерфейсе PAD+ AI:
   - Введите CLIENT_ID в поле "API Key" для GigaChat
   - Нажмите "Протестировать"
   - Если тест пройден успешно, провайдер будет активен

#### Важно:
- GigaChat использует автоматическое обновление токенов
- Токен действителен 30 минут
- Система автоматически обновляет токен при необходимости

### 3. Google Gemini

#### Требуется:
- API-ключ от Google AI Studio

#### Шаги:
1. Зайдите на [https://aistudio.google.com/](https://aistudio.google.com/)
2. Создайте API-ключ
3. Скопируйте ключ
4. В интерфейсе PAD+ AI:
   - Вставьте API-ключ в поле "API Key" для Gemini
   - Выберите желаемую модель (Gemini Pro, Gemini 1.5 Flash и т.д.)
   - Нажмите "Протестировать"
   - Если тест пройден успешно, провайдер будет активен

## Управление провайдерами через API

### Получение статуса провайдеров
```
GET /api/v1/llm/providers
```

### Установка конфигурации
```
POST /api/v1/llm/config
Content-Type: application/json

{
  "gigachat": {
    "enabled": true,
    "api_key": "ваш_api_key"
  },
  "gemini": {
    "enabled": true,
    "api_key": "ваш_api_key"
  },
  "openrouter": {
    "enabled": true,
    "api_key": "ваш_api_key",
    "model": "google/gemma-7b-it"
  }
}
```

## Устранение неполадок

### OpenRouter не проходит тест

1. Проверьте, что API-ключ действителен
2. Убедитесь, что у вас есть доступ к выбранной модели
3. Проверьте, не достигли ли вы лимита запросов
4. Убедитесь, что ключ не начинается или не заканчивается лишними символами

### GigaChat отображается как отключенный

1. Убедитесь, что вы ввели правильный AUTHORIZATION_KEY (Base64-кодированный)
2. Проверьте, что у вашего аккаунта есть доступ к GigaChat API
3. Убедитесь, что срок действия ключа не истек
4. Проверьте, что GIGACHAT_ENABLED установлен как true в настройках сервера

### Gemini не проходит тест

1. Проверьте, что API-ключ действителен
2. Убедитесь, что Google AI Studio активирован
3. Проверьте, что у вас есть доступ к выбранной модели Gemini
4. Убедитесь, что ключ не начинается или не заканчивается лишними символами

## Сессии и изоляция

### Индивидуальные настройки
Каждый пользователь может иметь свои собственные API-ключи:
1. Система создаст уникальную сессию
2. В рамках сессии можно настроить индивидуальные провайдеры
3. Ключи будут изолированы от других пользователей

### Использование в запросах
Для использования пользовательских провайдеров, передавайте `session_id` в заголовке:
```
X-Session-ID: ваш_session_id
```

## Безопасность

### Защита API-ключей
- Ключи не отображаются в открытом виде
- Ключи хранятся в защищённой сессии
- Ключи автоматически очищаются при истечении сессии
- Ключи не логгируются в системных журналах

### Рекомендации
- Не используйте одни и те же ключи на разных устройствах
- Регулярно обновляйте API-ключи
- Используйте ключи с ограниченными правами доступа
- Удаляйте неиспользуемые ключи

## Приоритеты провайдеров

Система использует следующий порядок приоритета:
1. **Пользовательские провайдеры** (из сессии)
2. **Системные провайдеры** (глобальные настройки)
3. **Fallback** (SQLite база с готовыми ответами)

## Заключение

После настройки одного или нескольких провайдеров, система будет использовать их в порядке приоритета. Убедитесь, что хотя бы один провайдер активен и проходит тест, чтобы система могла генерировать ответы для ваших запросов.
