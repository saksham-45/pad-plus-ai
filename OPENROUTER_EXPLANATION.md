# OpenRouter в системе PAD+ AI

## Что такое OpenRouter

OpenRouter - это платформа, которая предоставляет унифицированный доступ к различным моделям искусственного интеллекта от разных провайдеров через единый API-интерфейс.

### Основные особенности:
- **Многофункциональный API** для различных LLM (Large Language Models)
- Позволяет использовать модели от OpenAI, Google, Anthropic, Meta и других провайдеров через один интерфейс
- Поддерживает как платные, так и **бесплатные** модели (например, Google Gemma)

## Интеграция в PAD+ AI

### В конфигурации:
- В `render.yaml` OpenRouter включен по умолчанию: `OPENROUTER_ENABLED=true`
- Используется модель `google/gemma-7b-it` (бесплатная модель от Google)
- Может работать как альтернатива или резервный вариант при проблемах с другими провайдерами

### В коде системы:
- `backend/llm/provider_manager.py` - содержит реализацию OpenRouterProvider
- Поддерживает автоматическое тестирование и проверку доступности
- Интегрирован в каскад провайдеров (GigaChat → Google Gemini → OpenRouter → SQLite Fallback)

### Настройки в .env:
```env
OPENROUTER_API_KEY=ваш_api_ключ
OPENROUTER_ENABLED=true
OPENROUTER_MODEL=google/gemma-7b-it
```

## Преимущества OpenRouter

### 1. Бесплатные модели
- **Google Gemma 7B/9B** - полностью бесплатны
- **Meta Llama 3 8B** - часто доступна бесплатно
- Регулярно добавляются новые бесплатные модели

### 2. Разнообразие моделей
- Более 100 различных моделей от разных провайдеров
- От легковесных до мощных специализированных моделей
- Поддержка специализированных моделей (например, для кода, медицины, юриспруденции)

### 3. Простота интеграции
- Единый API для разных провайдеров
- Простая настройка через API-ключ
- Поддержка стандартных форматов запросов и ответов

### 4. Гибкость
- Можно легко переключаться между моделями
- Поддержка разных температур и параметров генерации
- Возможность использования как основного или резервного провайдера

## Настройка OpenRouter

### Получение API-ключа:
1. Зайдите на [https://openrouter.ai/keys](https://openrouter.ai/keys)
2. Создайте новый API-ключ
3. Скопируйте ключ и сохраните в безопасном месте

### В веб-интерфейсе:
- В разделе настроек провайдеров введите API-ключ в поле OpenRouter
- Выберите желаемую модель из выпадающего списка
- Нажмите "Протестировать" для проверки подключения

### В системе сессий:
- Каждый пользователь может использовать свои собственные OpenRouter API-ключи
- Ключи изолированы между сессиями пользователей
- Можно настроить как основной или резервный провайдер для конкретной сессии

## Сравнение с GigaChat

| Характеристика | OpenRouter | GigaChat |
|---|---|---|
| Доступность | Мировой рынок | Россия и СНГ |
| Языковая поддержка | Английский, русский и другие | Отличная поддержка русского |
| Бесплатные модели | Да (Gemma, Llama и др.) | Ограниченный доступ |
| Интеграция | Универсальный API | Российский API |
| Скорость ответа | Зависит от модели | Хорошая для русского языка |

## Роль в системе провайдеров

В системе PAD+ AI используется каскад провайдеров:
1. **GigaChat** (основной для российских пользователей)
2. **Google Gemini** (альтернативный)
3. **OpenRouter** (универсальный доступ к разным моделям)
4. **SQLite Fallback** (оффлайн-ответы)

Если основной провайдер недоступен, система автоматически переключается на следующий в списке.

## Безопасность и изоляция

- API-ключи OpenRouter изолированы между сессиями
- Ключи не передаются в ответах API
- Поддерживаются индивидуальные настройки для каждого пользователя
- Совместимость с системой безопасности PAD+ AI

## Заключение

OpenRouter предоставляет гибкую и мощную альтернативу для работы с различными LLM, особенно полезную для получения доступа к международным моделям и бесплатным вариантам, таким как Google Gemma. В системе PAD+ AI он служит важным резервным и альтернативным вариантом, обеспечивая надежность и доступность сервиса.
