# 🚀 Применение исправления RLS для обновления ключей

## Что было исправлено

1. **Модель API** - добавлена поддержка обновления самого ключа (api_key)
2. **Эндпоинт** - теперь может обновлять ключи с автоматическим шифрованием
3. **RLS политики** - добавлен WITH CHECK для UPDATE операций

## Применение исправления (1 минута)

### 🎯 Способ 1: Через Supabase Dashboard (Рекомендуется)

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard) и войдите
2. Выберите ваш проект
3. Перейдите в **SQL Editor** → **New query**
4. Откройте файл `scripts/fix_providers_rls.sql` и скопируйте весь SQL
5. Вставьте в SQL Editor и нажмите **Run**
6. Проверьте результат — должны появиться 4 новые политики

### 🎯 Способ 2: Через командную строку (PowerShell)

```powershell
cd scripts
.\fix_rls_policies.ps1
```

## Проверка

После применения исправления попробуйте:

1. **Добавить новый ключ:**
   - Откройте интерфейс
   - Перейдите в "Настройки" → "API ключи"
   - Нажмите "Добавить ключ"
   - Введите данные и сохраните

2. **Обновить существующий ключ:**
   - Нажмите на три точки рядом с ключом
   - Выберите "Изменить"
   - Обновите название, модель или сам ключ
   - Нажмите "Сохранить"

3. **Проверить логи:**
   ```
   ✅ Key updated successfully: key_id=...
   ```

## Что если ошибка сохраняется?

**Проблема:** Ошибка `permission denied` или `new row violates row-level security policy`

**Решение:**
1. Убедитесь, что вы скопировали ВСЕ SQL из `fix_providers_rls.sql`
2. Проверьте, что запрос выполнился без ошибок (должна быть зеленая галка)
3. Откройте Supabase Dashboard → **Authentication** → **Policies**
4. Убедитесь, что есть 4 политики:
   - Users can view own keys (SELECT)
   - Users can insert own keys (INSERT)
   - Users can update own keys (UPDATE)
   - Users can delete own keys (DELETE)

Если политик нет, скопируйте SQL еще раз и выполните.
