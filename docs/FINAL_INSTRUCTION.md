- Backend стабильно работает на порту 8080

### **Шаг 2: Регистрация пользователя**
```bash
curl -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"your@email.com","password":"your_password","name":"Your Name"}'
```

### **Шаг 3: Вход в систему**
```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"your@email.com","password":"your_password"}'
```

### **Шаг 4: Работа с API ключами**
```bash
# Получение ключей
curl -X GET http://localhost:8080/api/v1/keys \
  -H "Authorization: Bearer YOUR_TOKEN"

# Создание ключа
curl -X POST http://localhost:8080/api/v1/keys \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"My Key","provider":"gigachat","api_key":"your_api_key"}'
```

## 📋 **ДОСТУПНЫЕ ЭНДПОИНТЫ:**

### **Авторизация:**
- `POST /api/v1/auth/register` - Регистрация
- `POST /api/v1/auth/login` - Вход
- `POST /api/v1/auth/refresh` - Обновление токена
- `GET /api/v1/auth/me` - Профиль пользователя

### **API Ключи:**
- `GET /api/v1/keys` - Получение ключей
- `POST /api/v1/keys` - Создание ключа

### **Система:**
- `GET /` - Статус backend
- `GET /api/v1/providers` - Список провайдеров
- `GET /api/v1/health` - Проверка здоровья
- `GET /api/v1/metrics/system` - Метрики системы
