# Развертывание PAD+ AI на Render

## Подготовка к развертыванию

### 1. Генерация секретных ключей

Перед развертыванием необходимо сгенерировать следующие ключи:

```bash
# Генерация ключа шифрования
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

# Генерация соли для шифрования
python -c "import base64, os; print(base64.urlsafe_b64encode(os.urandom(32)).decode())"

# Генерация CSRF секретного ключа
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

### 2. Настройка переменных окружения

В панели управления Render необходимо добавить следующие переменные окружения:

#### Обязательные переменные:
- `ENCRYPTION_KEY` - сгенерированный ключ шифрования
- `ENCRYPTION_SALT` - сгенерированная соль
- `CSRF_SECRET_KEY` - сгенерированный CSRF ключ
- `SUPABASE_URL` - URL вашего Supabase проекта
- `SUPABASE_KEY` - ключ вашего Supabase проекта
- `GIGACHAT_AUTH_KEY` - ваш GigaChat authorization key

#### Опциональные переменные:
- `DEBUG` - `false` для production
- `LOG_LEVEL` - `info` для production
- `FRONTEND_URL` - будет автоматически установлен Render

### 3. Настройка базы данных

В панели управления Render:
1. Создайте PostgreSQL базу данных с именем `pad-plus-db`
2. Создайте Redis сервис с именем `pad-plus-redis`

### 4. Развертывание

1. Зайдите в ваш Render аккаунт
2. Создайте новый сервис из GitHub репозитория
3. Выберите файл `render.yaml` в качестве конфигурации
4. Разверните приложение

## Структура сервиса Render

### Основной сервис (Web)
- **Имя**: `pad-plus-ai`
- **Тип**: Web
- **Рантайм**: Python 3.11.0
- **План**: Standard
- **Порт**: 8000

### База данных
- **Имя**: `pad-plus-db`
- **Тип**: PostgreSQL
- **План**: Standard
- **База данных**: `padplus`
- **Пользователь**: `padplus`

### Redis кэш
- **Имя**: `pad-plus-redis`
- **Тип**: Redis
- **План**: Starter

## Процесс сборки

Сборка происходит в следующем порядке:
1. Установка Python зависимостей (`requirements.txt`)
2. Установка Node.js зависимостей (`frontend/package.json`)
3. Сборка frontend (`npm run build`)
4. Применение миграций базы данных (`python scripts/apply_migrations.py`)

## Запуск приложения

Приложение запускается командой:
```bash
python backend/main.py
```

## Возможные проблемы и решения

### 1. Проблемы с ChromaDB

ChromaDB может не работать корректно в Render из-за ограничений файловой системы. Рекомендуется:

1. Использовать ChromaDB в режиме HTTP
2. Или переключиться на другую векторную базу данных (например, Supabase Vector)

### 2. Проблемы с памятью

Если приложение использует слишком много памяти:
1. Перейдите на более высокий план Render
2. Оптимизируйте использование памяти в коде
3. Используйте Redis для кэширования

### 3. Проблемы с производительностью

Для улучшения производительности:
1. Используйте CDN для статических файлов
2. Оптимизируйте запросы к базе данных
3. Включите кэширование ответов

## Мониторинг

Приложение включает встроенный мониторинг:
- **Health check**: `/health`
- **Metrics**: `/metrics` (Prometheus формат)
- **Logs**: Доступны в панели управления Render

## Безопасность

1. Все API ключи должны быть зашифрованы
2. Используйте HTTPS в production
3. Включите CSRF защиту
4. Регулярно обновляйте зависимости

## Обновление

Для обновления приложения:
1. Сделайте коммит в GitHub
2. Render автоматически запустит сборку
3. Приложение будет обновлено автоматически

## Откат изменений

Если развертывание прошло неудачно:
1. Откройте панель управления Render
2. Перейдите в раздел "Deployments"
3. Откатитесь к предыдущей успешной версии

## Контакты поддержки

При возникновении проблем:
1. Проверьте логи в панели Render
2. Проверьте health check эндпоинт
3. Обратитесь в поддержку Render
