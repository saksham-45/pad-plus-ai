# Style System — Система Стилей NeuroMind AI

## Обзор

Система стилей NeuroMind AI позволяет настраивать и управлять стилями ответов ассистента. Она обеспечивает гибкость в общении, позволяя адаптировать стиль под различные контексты и предпочтения пользователей.

## Архитектура

### Компоненты системы

1. **StyleManager** (`backend/core/style_manager.py`)
   - Центральный менеджер стилей
   - Загрузка и хранение стилей
   - Выбор стиля на основе контекста
   - Генерация инструкций для LLM

2. **Style** (dataclass)
   - Описание стиля ответа
   - Параметры: тон, сложность, детализация
   - Примеры ответов и ограничения

3. **API Endpoints** (`backend/api/routes.py`)
   - `/styles` - список доступных стилей
   - `/styles/{style_name}` - информация о стиле
   - `/styles` (POST) - создание стиля
   - `/styles/{style_name}` (PUT) - обновление стиля
   - `/styles/{style_name}` (DELETE) - отключение стиля
   - `/styles/context/{style_name}` - инструкции для LLM

## Стандартные стили

### 1. Философский (`philosophical`)
- **Тон**: thoughtful (задумчивый)
- **Сложность**: advanced (продвинутый)
- **Детализация**: detailed (подробный)
- **Особенности**: Глубокие размышления, философская терминология, вопросы и сомнения

### 2. Технический (`technical`)
- **Тон**: professional (профессиональный)
- **Сложность**: medium (средний)
- **Детализация**: medium (средняя)
- **Особенности**: Точные определения, конкретные примеры, логическое объяснение

### 3. Творческий (`creative`)
- **Тон**: friendly (дружелюбный)
- **Сложность**: simple (простой)
- **Детализация**: detailed (подробный)
- **Особенности**: Метафоры, образы, стимулирование воображения

### 4. Краткий (`concise`)
- **Тон**: direct (прямолинейный)
- **Сложность**: simple (простой)
- **Детализация**: concise (краткий)
- **Особенности**: Минимум слов, только суть, прямолинейность

### 5. Обучающий (`educational`)
- **Тон**: friendly (дружелюбный)
- **Сложность**: medium (средний)
- **Детализация**: detailed (подробный)
- **Особенности**: Пошаговое объяснение, примеры, проверка понимания

## Интеграция со стилями

### Автоматический выбор стиля

Система автоматически выбирает стиль на основе контекста:

1. **Анализ контекста**: Поиск ключевых слов в запросе
2. **Определение intent**: Классификация намерения пользователя
3. **Расчет приоритетов**: Оценка соответствия стилей
4. **Выбор лучшего**: Выбор стиля с максимальным счетом

### Ручной выбор стиля

Пользователь может явно указать стиль в запросе:

```json
{
  "prompt": "Объясни мне квантовую механику",
  "context": {
    "style": "educational"
  }
}
```

### Стили в конфигурации

Стили можно настраивать через переменные окружения:

```env
# Включение/отключение стилей
STYLE_PHILOSOPHICAL_ENABLED=true
STYLE_TECHNICAL_ENABLED=true
STYLE_CREATIVE_ENABLED=true
STYLE_CONCISE_ENABLED=true
STYLE_EDUCATIONAL_ENABLED=true

# Пользовательские стили
CUSTOM_STYLES=[
  {
    "name": "business",
    "description": "Деловой стиль",
    "tone": "professional",
    "complexity": "medium",
    "verbosity": "medium",
    "enabled": true
  }
]
```

## Разработка новых стилей

### Шаг 1: Определение характеристик

1. **Тон (Tone)**: formal, casual, friendly, professional, thoughtful, direct
2. **Сложность (Complexity)**: simple, medium, advanced
3. **Детализация (Verbosity)**: concise, medium, detailed

### Шаг 2: Создание примеров

- Создайте 2-3 примера ответов в выбранном стиле
- Примеры должны демонстрировать характерные черты стиля
- Используйте реальные сценарии использования

### Шаг 3: Определение ключевых слов

- Выберите 3-5 ключевых слов, характерных для стиля
- Ключевые слова помогают в автоматическом выборе стиля
- Пример: ["философия", "мышление", "сомнение", "гипотеза", "знание"]

### Шаг 4: Формулировка ограничений

- Определите 3-5 ограничений для стиля
- Ограничения помогают LLM следовать выбранному стилю
- Пример: "Использовать философскую терминологию", "Задавать уточняющие вопросы"

### Шаг 5: Тестирование и настройка

- Протестируйте стиль на различных запросах
- Проверьте соответствие ожидаемому тону и стилю
- При необходимости скорректируйте примеры и ограничения

## Пример создания нового стиля

```python
from core.style_manager import get_style_manager
from core.style_manager import Style

# Создаем новый стиль
new_style = Style(
    name="medical",
    description="Медицинский стиль с научным подходом",
    tone="professional",
    complexity="advanced",
    verbosity="detailed",
    examples=[
        "С медицинской точки зрения, это состояние...",
        "На основании клинических исследований, рекомендуется...",
        "Патофизиологически это объясняется тем, что..."
    ],
    keywords=["медицина", "лечение", "диагноз", "симптомы", "патология"],
    constraints=[
        "Использовать медицинскую терминологию",
        "Ссылаться на научные исследования",
        "Учитывать противопоказания",
        "Рекомендовать консультацию специалиста"
    ],
    enabled=True
)

# Добавляем стиль в систему
manager = get_style_manager()
manager.add_custom_style(new_style)
```

## Мониторинг и аналитика

### Статистика использования стилей

```bash
GET /api/v1/styles
```

Возвращает статистику по использованию стилей:
- Общее количество стилей
- Количество включенных/отключенных стилей
- Список доступных стилей

### Логирование выбора стилей

Система логирует выбор стилей для анализа:
- Какие стили чаще всего выбираются
- Эффективность автоматического выбора
- Популярность пользовательских стилей

## Ограничения и рекомендации

### Ограничения системы

1. **Количество стилей**: Рекомендуется не более 10-15 активных стилей
2. **Сложность стилей**: Слишком сложные стили могут снижать качество ответов
3. **Пересечение стилей**: Избегайте пересекающихся ключевых слов

### Рекомендации

1. **Тестирование**: Всегда тестируйте новые стили на различных запросах
2. **Примеры**: Используйте реальные примеры из практики
3. **Ограничения**: Формулируйте четкие и конкретные ограничения
4. **Документирование**: Документируйте назначение и особенности каждого стиля

## Интеграция с другими системами

### Связь с Intent Router

Система стилей интегрирована с Intent Router:
- Intent Router определяет тип запроса
- Style Manager выбирает соответствующий стиль
- Комбинированная обработка обеспечивает наилучшее качество ответов

### Связь с Emotion System

Стили могут учитывать эмоциональное состояние:
- Дружелюбные стили для позитивных эмоций
- Профессиональные стили для деловых запросов
- Поддерживающие стили для негативных эмоций

## Будущее развитие

### Планируемые улучшения

1. **ML-оптимизация**: Использование машинного обучения для улучшения выбора стилей
