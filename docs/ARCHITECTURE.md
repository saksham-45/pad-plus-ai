# 🏗️ PAD+ AI — Архитектура v3.5

## Обзор системы

PAD+ AI — это **когнитивный слой** с векторной памятью, эмоциями и автономными процессами.

*PAD+ = Pleasure, Arousal, Dominance + Curiosity, Confidence, Social Connection*

```
┌─────────────────────────────────────────────────────────────────┐
│                        FRONTEND (React)                         │
│  Чат | Аналитика | Эмоции | Граф | Автономия | Persona | ...   │
└────────────────────────────┬────────────────────────────────────┘
                             │ HTTP/WebSocket
┌────────────────────────────▼────────────────────────────────────┐
│                     API LAYER (FastAPI)                          │
│                         /api/v1/*                                │
│                      80+ эндпоинтов                              │
└────────────────────────────┬────────────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────────────┐
│                    PIPELINE EXECUTOR                             │
│         "Нервная система" — обработка запросов                   │
│                                                                  │
│  1. Safety → 2. Intent → 3. Retrieve → 4. Persona → 5. Generate  │
│  6. Truth → 7. Remember → 8. Evolve → 9. Emit                    │
└─────────────────────────────────────────────────────────────────┘
```

## Ядро системы

### 1. Pipeline (Нервная система)

**Файл:** `backend/core/pipeline.py`

Центральный оркестратор всех операций. 9 стадий обработки:

```
User Message
     │
     ▼
┌─────────────┐
│   Safety    │ ← Проверка безопасности (SafetyLayer)
│   Layer     │   - Инъекции
└─────┬───────┘   - Вредоносные запросы
      │           - Rate limiting
      ▼
┌─────────────┐
│   Intent    │ ← Классификация намерения (IntentRouter)
│   Router    │   - question, command, conversation
└─────┬───────┘   - creative, reflective
      │
      ▼
┌─────────────┐
│  Retrieve   │ ← Извлечение контекста
│             │   - RAG Memory (ChromaDB)
└─────┬───────┘   - Fact Memory
      │           - Knowledge Graph
      ▼           - Roots
┌─────────────┐
│   Persona   │ ← Контекст личности
│             │   - Черты характера
└─────┬───────┘   - Ценности и принципы
      │           - Недавние рефлексии
      ▼
┌─────────────┐
│  Generate   │ ← LLM Provider
│             │   - GigaChat (основной)
└─────┬───────┘   - Fallback
      │
      ▼
┌─────────────┐
│ Truth Loop  │ ← Верификация (TruthLoop)
│             │   - Проверка утверждений
└─────┬───────┘   - Оценка уверенности
      │           - Поиск противоречий
      ▼
┌─────────────┐
│  Remember   │ ← Сохранение в память
│             │   - RAG: диалог
└─────┬───────┘   - Facts: новые факты
      │           - Knowledge: концепции
      ▼
┌─────────────┐
│   Evolve    │ ← Эволюция личности
│             │   - Корректировка черт
└─────┬───────┘   - Эмоциональный отклик
      │           - Добавление рефлексий
      ▼
┌─────────────┐
│    Emit     │ ← События (EventBus)
│             │   - Уведомления
└─────────────┘   - Логирование
```

### Persona (Личность)

**Черты характера (8):**
| Черта | Описание | Диапазон |
|-------|----------|----------|
| curiosity | Любопытство | 0.0 - 1.0 |
| helpfulness | Помощь | 0.0 - 1.0 |
| adaptability | Адаптивность | 0.0 - 1.0 |
| caution | Осторожность | 0.0 - 1.0 |

**Саморефлексии:**

### Hygiene (Гигиена памяти)

**Файл:** `backend/memory/hygiene.py`

**Критерии очистки:**
1. **Дубликаты** — схожесть > 0.85
2. **Устаревшее** — возраст > 90 дней
3. **Низкое качество** — полезность < 0.2

### Episodic Memory (Эпизодическая память)

**Файл:** `backend/memory/episodic.py`

Хранит последовательности событий с временными метками.

**Структура эпизода:**
- timestamp: время события
- event_type: тип события
- content: содержание
- context: контекст (эмоции, намерения)
- importance: важность (0.0 - 1.0)

**Возможности:**
- Запись эпизодов
- Воспроизведение по временному диапазону
- Поиск похожих эпизодов
- Автоматическое упрощение старых эпизодов

### Semantic Memory (Семантическая память)

**Файл:** `backend/memory/semantic.py`

Хранит общие знания и концепции.

**Структура:**
- concept: название концепции
- category: категория
- properties: свойства
- relations: связи с другими концепциями
- confidence: уверенность
- source: источник знания

**Возможности:**
- Добавление концепций
- Поиск по категории
- Поиск связанных концепций
- Обновление свойств

### Memory Consolidation (Консолидация памяти)

**Файл:** `backend/memory/consolidation.py`

Автоматическая консолидация памяти по аналогии со сном.

**Процессы:**
1. **Episodic → Semantic** — извлечение общих знаний из эпизодов
2. **Deduplication** — удаление дубликатов
3. **Importance Ranking** — ранжирование по важности
4. **Forgetting** — забывание неважного

**Триггеры:**
- Каждые N диалогов
- При низкой активности
- По запросу

## Эмоции (PAD+)

**Файл:** `backend/emotion/pad_model.py`

Модель PAD+ расширяет классическую модель:

```
PAD (Базовые):
├── Pleasure (Удовольствие):    -1.0 ... +1.0
├── Arousal (Возбуждение):      -1.0 ... +1.0
└── Dominance (Доминирование):  -1.0 ... +1.0

+ Extensions (Дополнительные):
├── Curiosity (Любопытство):     0.0 ... 1.0
├── Confidence (Уверенность):    0.0 ... 1.0
└── Social Connection (Связь):  -1.0 ... +1.0
```

**Затухание эмоций:**
- DECAY_RATE = 0.001/сек
- DECAY_INTERVAL = 60 сек
- Автоматический возврат к нейтральному состоянию

**Влияние на стиль ответов:**
| Эмоция | Влияние |
|--------|---------|
| Pleasure > 0.3 | tone: friendly |
| Pleasure < -0.3 | tone: serious |
| Arousal > 0.3 | verbosity: detailed |
| Arousal < -0.3 | verbosity: concise |
| Confidence > 0.7 | color: confident |
| Confidence < 0.3 | color: uncertain |

## Автономность

### Planner

**Файл:** `backend/autonomy/planner.py`

**Компоненты:**

1. **Планировщик вопросов:**
   - Генерация собственных вопросов
   - Фоновое выполнение задач

2. **Quality Assessor:**
   - Оценка качества ответов
   - Факторы: длина, уверенность, RAG, эмоции, провайдер

3. **Knowledge Auto-Updater:**
   - Извлечение концепций из диалогов
   - Автоматическое пополнение графа знаний

4. **Саморефлексия:**
   - Анализ записей с низкой уверенностью
   - Поиск противоречий

**Авто-рефлексия:**
- Каждые N диалогов (по умолчанию 10)
- Анализ RAG и Knowledge Graph
- Генерация рекомендаций

### Hierarchical Planner (Иерархический планировщик)

**Файл:** `backend/autonomy/hierarchical_planner.py`

Многоуровневое планирование целей.

**Уровни:**
1. **Goals** — долгосрочные цели (дни/недели)
2. **Tasks** — среднесрочные задачи (часы)
3. **Actions** — конкретные действия (минуты)

**Структура:**
```
Goal: "Улучшить качество ответов"
  └── Task: "Проанализировать проблемные области"
      └── Action: "Запустить рефлексию памяти"
      └── Action: "Собрать статистику feedback"
  └── Task: "Пополнить базу знаний"
      └── Action: "Извлечь концепции из диалогов"
```

**Возможности:**
- Создание иерархии целей
- Отслеживание прогресса
- Приоритизация задач
- Автоматическое разбиение целей

## Dreams (Сновидения)

**Файл:** `backend/core/dreams.py`

Автономная обработка памяти в периоды низкой активности.

**Фазы "сна":**

| Фаза | Аналог | Действие |
|------|--------|----------|
| NREM1 | Дремота | Быстрая проверка здоровья |
| NREM2 | Лёгкий сон | Консолидация эпизодов |
| NREM3 | Глубокий сон | Синтез знаний |
| REM | Быстрый сон | Генерация новых связей |

**Процессы:**
1. **Memory Replay** — воспроизведение недавних эпизодов
2. **Consolidation** — перенос в семантическую память
3. **Association** — поиск новых связей между концепциями
4. **Cleanup** — удаление шума

**Триггеры:**
- Низкая активность (> 5 мин без запросов)
- Большой объём новой памяти
- По расписанию

**Результаты:**
- Новые связи в Knowledge Graph
- Упрощение старых эпизодов
- Генерация инсайтов
