# Структура проекта

## Основные папки

- `backend/` — Python backend проекта, включая API, бизнес-логику, runtime и интеграции.
- `frontend/` — клиентская часть на React/Vite для взаимодействия с пользователем.
- `tests/` — основной набор тестов для проекта.
- `tests/moved_from_root/` — перенесённые из корня тесты, чтобы сохранить их как часть тестовой истории без загромождения корня.
  - Примечание: часть этих файлов представляет собой legacy-скрипты и требует запущенного backend на `localhost:8080` и корректных переменных окружения.
- `docs/` — документация проекта, инструкции по развертыванию, архитектуре и безопасности.
- `docs/archive/` — архив старых отчётов, устаревших заметок и ранее использовавшихся документов.
- `scripts/` — утилиты и вспомогательные скрипты, включая `cleanup_repo.ps1`.
- `archive/` — временный/кладовой архив для удаления чувствительных данных и больших артефактов. Этот каталог исключён из git через `.gitignore`.

## Что изменено

- Корневые тесты перемещены в `tests/moved_from_root/`.
- Старые отчётные документы перемещены в `docs/archive/`.
- Новый README теперь содержит ссылку на документацию по структуре проекта.

## Как ориентироваться

- Для развертывания и быстрых инструкций используйте `README.md`.
- Для подробной документации по проекту и архитектуре используйте `docs/`.
- Для архивных и устаревших материалов используйте `docs/archive/`.

## Рекомендации

- Не размещайте новые логические файлы на уровне корня, если они относятся к бэкенду, фронтенду или документации.
- Новые тесты добавляйте в `tests/`, а не в корень проекта.
- Старые отчёты оставляйте в `docs/archive/` для справки и истории изменений.
