# 📚 Подробное руководство по применению миграций базы данных

## 🔍 Что произошло?

В системе PAD+ AI была исправлена навигация, и теперь вкладка "Документы" доступна в интерфейсе. Однако для работы с документами, историей диалогов и настройками пользователя необходимо создать соответствующие таблицы в базе данных Supabase.

## 🎯 Что нужно сделать?

Применить 5 SQL-миграций к базе данных Supabase через веб-интерфейс.

---

## 📖 Пошаговая инструкция

### Шаг 1: Откройте Supabase Dashboard

1. Перейдите по ссылке: **https://app.supabase.com**
2. Войдите в свой аккаунт (если еще не вошли)
3. В списке проектов найдите ваш проект PAD+ AI и кликните на него

### Шаг 2: Перейдите в SQL Editor

1. В левом меню выберите **"SQL Editor"** (иконка с символом `</>`)
2. Откроется редактор SQL-запросов

### Шаг 3: Подготовьте SQL-файлы

Откройте следующие файлы в вашем проекте (они находятся в папке `backend/database/migrations/`):

1. `001_initial_schema.sql`
2. `002_rls_policies.sql`
3. `003_fix_rls_policies.sql`
4. `004_user_settings_and_dialogs.sql`
5. `005_documents_and_collections.sql`

### Шаг 4: Выполните миграции по порядку

#### Миграция 1: 001_initial_schema.sql

1. Откройте файл `backend/database/migrations/001_initial_schema.sql`
2. Скопируйте **всё содержимое** файла (Ctrl+A, Ctrl+C)
3. Вставьте в SQL Editor в Supabase (Ctrl+V)
4. Нажмите кнопку **"Run"** (или Ctrl+Enter)
5. Дождитесь сообщения об успешном выполнении

#### Миграция 2: 002_rls_policies.sql

1. Откройте файл `backend/database/migrations/002_rls_policies.sql`
2. Скопируйте содержимое
3. Вставьте в SQL Editor (очистите предыдущий запрос)
4. Нажмите **"Run"**
5. Дождитесь успеха

#### Миграция 3: 003_fix_rls_policies.sql

1. Откройте файл `backend/database/migrations/003_fix_rls_policies.sql`
2. Скопируйте содержимое
3. Вставьте в SQL Editor
4. Нажмите **"Run"**
5. Дождитесь успеха

#### Миграция 4: 004_user_settings_and_dialogs.sql

1. Откройте файл `backend/database/migrations/004_user_settings_and_dialogs.sql`
2. Скопируйте содержимое
3. Вставьте в SQL Editor
4. Нажмите **"Run"**
5. Дождитесь успеха

#### Миграция 5: 005_documents_and_collections.sql

1. Откройте файл `backend/database/migrations/005_documents_and_collections.sql`
2. Скопируйте содержимое
3. Вставьте в SQL Editor
4. Нажмите **"Run"**
5. Дождитесь успеха

### Шаг 5: Проверьте результат

1. В левом меню выберите **"Table Editor"** (иконка таблицы)
2. Вы должны увидеть новые таблицы:
   - `documents`
   - `document_collections`
   - `dialogs`
   - `user_settings`
   - и другие

### Шаг 6: Перезапустите backend

1. Остановите backend сервер (Ctrl+C в терминале)
2. Запустите заново: `python backend/main.py` или через `start.bat`
3. Проверьте, что ошибки с отсутствующими таблицами больше не появляются

## ❓ Что делать, если что-то пошло не так?

### Ошибка: "relation already exists"

Если вы видите ошибку, что таблица уже существует, значит миграция уже была применена ранее. Пропустите эту миграцию.

### Ошибка: "syntax error at or near"

Проверьте, что вы скопировали весь SQL-код без обрезаний. Убедитесь, что в редакторе нет лишних символов.

### Ошибка: "permission denied"

Убедитесь, что вы используете проект с правами администратора. В Supabase это обычно работает по умолчанию.

### Таблицы не появились

1. Проверьте, что все миграции выполнены успешно (зеленая галочка)
2. Обновите страницу Table Editor (F5)
3. Убедитесь, что вы в правильном проекте Supabase
