# Обзор архитектуры системы PAD+ AI v3.5

---

## 1. Как реально работает система

Система представляет собой монолитный FastAPI приложение с единым центральным пайплайном обработки запросов.

**Реальный поток:**
Любой запрос пользователя проходит строго последовательный пайплайн из 18 стадий. На самой первой стадии перед всеми другими компонентами работает **X-Ray Brain** который принимает абсолютно все стратегические решения: какую стратегию использовать, какую LLM вызвать, использовать ли память, какая глубина анализа нужна, нужна ли верификация.

После принятия решения Brain передаёт управление линейному исполнителю PipelineExecutor который просто выполняет предписанные шаги. Никаких других центров принятия решений в системе нет.

Система работает полностью синхронно на каждый запрос, нет параллелизма стадий. Каждая стадия имеет собственный try/except блок и при ошибке просто деградирует с fallback логикой без остановки всего пайплайна.

---

## 2. Где принимаются решения

✅ **Единственный абсолютный центр принятия решений:**
> **XRayBrain** (файл `backend/core/xray/brain.py`)

Все остальные компоненты являются чистыми исполнителями. Ни один другой модуль не принимает решений, не выбирает стратегии и не меняет поток выполнения.

**Полномочия Brain:**
- Выбирает один из 6 типов стратегий обработки
- Назначает LLM модель под сложность запроса
- Определяет какие из 7 систем памяти использовать
- Устанавливает глубину анализа от 1 до 5
- Решает нужна ли верификация ответа
- Определяет тон ответа

При любой ошибке Brain всегда падает на простую стратегию.

## 4. Главные проблемы архитектуры

### ⚠️ Критические проблемы:

1. **Единственная точка отказа** - X-Ray Brain является единственным местом принятия всех решений. Любая ошибка в Brain ломает всю систему.

2. **Тихое поглощение ошибок** - абсолютно все стадии пайплайна имеют независимые try/except блоки. Ошибки просто логируются и система продолжает работать в деградированном режиме без какого-либо уведомления пользователя.

3. **Конфликт памяти** - в системе реализовано 7 независимых систем памяти с одинаковым интерфейсом, не имеющие централизованного управления и транзакционности.

4. **Нет глобального состояния** - Fail Strategy работает только на уровне отдельного пайплайна, нет глобального состояния деградации системы.

5. **Отсутствие отката решений** - решения принятые Brain никогда не пересматриваются, нет механизма обратной связи.

## 5. Уровень системы

✅ **Уровень: Functional (2 из 5)**

**Оценка:**
- ❌ Не Primitive: система имеет структуру, пайплайн и отдельные компоненты
- ✅ Functional: работает линейный поток, есть централизованное принятие решений, реализованы основные функции
- ❌ Не Adaptive: нет реального обучения, нет самоулучшения, все решения основаны на статических правилах
- ❌ Не Autonomous: нет самоназначения целей, нет долгосрочного планирования
- ❌ Не AGI

**Реальные возможности:**
Система может классифицировать запросы, выбирать подходящую стратегию, использовать несколько типов памяти, генерировать и верифицировать ответы. Все решения основаны на жестко закодированных правилах и ключевых словах.
