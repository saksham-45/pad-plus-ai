# 🧠 NeuroMind AI - Детальный план архитектуры

## 📋 Обзор системы

NeuroMind AI - это автономный ИИ-агент с многослойной памятью, эмоциональной моделью и системой саморефлексии. Система реализует концепцию PAD+ (Pleasure, Arousal, Dominance + Curiosity, Confidence, Social Connection) и использует философское ядро ANTI_DIRECTIVE для постоянного сомнения и проверки.

## 🏗️ Архитектурные слои

### 1. Ядро системы (Core)

#### 1.1. ANTI_DIRECTIVE
- **Файл**: `backend/core/anti_directive.py`
- **Назначение**: Философское ядро системы, обеспечивающее принципы сомнения и проверки
- **Функции**:
  - Проверка целостности ядра
  - Фильтрация знаний через принцип "не закрепляй знания, сомневайся, проверяй"
  - Блокировка категоричных утверждений

#### 1.2. Pipeline Executor (Пайплайн)
- **Файл**: `backend/core/pipeline.py`
- **Назначение**: Основной оркестратор обработки запросов
- **Фазы обработки**:
  1. Safety → Проверка безопасности
  2. Intent → Классификация намерения
  3. Retrieve → Извлечение контекста (RAG, факты, знания)
  4. Persona → Контекст личности
  5. Generate → Генерация ответа
  6. Truth → Верификация
  7. Remember → Сохранение в память
  8. Emit → События

#### 1.3. Intent Router
- **Файл**: `backend/core/intent_router.py`
- **Назначение**: Классификация намерений пользователя
- **Типы намерений**:
  - CHAT_GENERAL - обычный диалог
  - CHAT_RAG - нужен RAG контекст
  - CLARIFY - нужно уточнение
  - TASK_CREATE - создание задачи
  - TASK_EXECUTE - выполнение задачи
  - MEMORY_WRITE - запись в память
  - KNOWLEDGE_QUERY - вопрос к графу знаний
  - SYSTEM_ADMIN - настройки/отладка

#### 1.4. Truth Loop
- **Файл**: `backend/core/truth_loop.py`
- **Назначение**: Верификация утверждений и проверка достоверности
- **Функции**:
  - Извлечение утверждений из текста
  - Проверка против памяти
  - Внутренняя проверка согласованности
  - Назначение уверенности
  - Обновление памяти и графа

#### 1.5. Safety Layer
- **Файл**: `backend/core/safety_layer.py`
- **Назначение**: Защита системы от различных угроз
- **Функции**:
  - Фильтрация опасных запросов
  - Проверка на манипуляции
  - Rate limiting
  - Ограничение автономии
  - Санитизация ввода

#### 1.6. Meta Controller
- **Файл**: `backend/core/meta_controller.py`
- **Назначение**: Управление стратегиями обработки запросов
- **Функции**:
  - Оценка когнитивной нагрузки
  - Принятие решений о стратегиях
  - Управление подсистемами
  - Координация обучения и адаптации

#### 1.7. Event Bus
- **Файл**: `backend/core/event_bus.py`
- **Назначение**: Внутренняя коммуникация между модулями
- **Типы событий**:
  - dialogue.finished
  - memory.updated
  - emotion.changed
  - knowledge.new_entity
  - autonomy.task_created
  - truth.claim_verified

### 2. Память (Memory)

#### 2.1. RAG Memory (v3.0)
- **Файл**: `backend/memory/rag.py`
- **Назначение**: Семантическая память с ChromaDB
- **Функции**:
  - Классификация тем диалогов
  - Извлечение сущностей и связей
  - LLM-суммаризация
  - Гибридный поиск с ранжированием
  - Классификация: техническое, философское, личное, образовательное, творческое, аналитическое, бытовое

#### 2.2. Fact Memory
- **Файл**: `backend/memory/fact_memory.py`
- **Назначение**: Структурированные факты (subject-predicate-object)
- **Функции**:
  - Хранение атомарных фактов
  - Автопоиск противоречий
  - Поиск по субъекту/объекту
  - Поддержка ревизии фактов

#### 2.3. Episodic Memory
- **Файл**: `backend/memory/episodic.py`
- **Назначение**: Эпизодическая память с контекстом ситуаций
- **Функции**:
  - Хранение диалогов с контекстом
  - Эмоциональный след эпизода
  - Связи между эпизодами
  - Важность для консолидации

#### 2.4. Semantic Memory
- **Файл**: `backend/memory/semantic.py`
- **Назначение**: Семантическая память с типизированным хранением
- **Типы знаний**:
  - DECLARATIVE - факты
  - PROCEDURAL - навыки
  - CONCEPTUAL - концепции
  - METACOGNITIVE - о себе

#### 2.5. Roots Memory
- **Файл**: `backend/memory/roots.py`
- **Назначение**: Фундаментальные принципы (неизменяемые)
- **Категории**:
  - Philosophy - философские принципы
  - Cognitive - когнитивные принципы
  - Communication - коммуникативные принципы
  - Ethics - этические принципы
  - Identity - факты об идентичности

#### 2.6. Persona Memory
- **Файл**: `backend/memory/persona.py`
- **Назначение**: Личность системы
- **Функции**:
  - Хранение черт характера
  - История саморефлексий
  - Предпочтения стиля
  - Отношения с пользователями

#### 2.7. Memory Consolidation
- **Файл**: `backend/memory/consolidation.py`
- **Назначение**: Консолидация памяти между слоями
- **Процессы**:
  - Episodic → Semantic (извлечение общих знаний)
  - Semantic → Roots (стабилизация важных знаний)
  - RAG → Semantic (извлечение фактов)
  - Обновление связей между знаниями

#### 2.8. Memory Hygiene
- **Файл**: `backend/memory/hygiene.py`
- **Назначение**: Поддержание чистоты памяти
- **Функции**:
  - Удаление дубликатов
  - Чистка устаревшего
  - Оценка полезности
  - Забывание ненужного

### 3. Эмоции (Emotion)

#### 3.1. PAD+ Model
- **Файл**: `backend/emotion/pad_model.py`
- **Назначение**: Модель эмоций PAD+
- **Параметры**:
  - Pleasure (Удовольствие): -1 до +1
  - Arousal (Возбуждение): -1 до +1
  - Dominance (Доминирование): -1 до +1
  - Curiosity (Любопытство): 0 до 1
  - Confidence (Уверенность): 0 до 1
  - Social Connection (Социальная связь): -1 до +1
- **Функции**:
  - Затухание эмоций
  - Влияние на стиль общения
  - Сохранение в файл

## 🔄 Архитектурные паттерны

### 1. Event-Driven Architecture
- Используется EventBus для внутренней коммуникации
- Асинхронная обработка событий
- Модульность через события

### 2. Multi-Layer Memory Architecture
- Иерархия: Shells → Soil → Roots
- Каждый слой отвечает за свой тип информации
- Механизмы консолидации между слоями

### 3. Cognitive Architecture
- Мета-когнитивный контроль
- Саморефлексия и адаптация
- Постоянная проверка и пересмотр

### 4. Emotional Intelligence
- PAD+ модель эмоций
- Влияние на стиль общения
- Автоматическое затухание

## 📅 Жизненный цикл запроса

1. **Получение** - WebSocket или HTTP
2. **Проверка безопасности** - Safety Layer
3. **Классификация намерения** - Intent Router
4. **Извлечение контекста** - RAG, Facts, Knowledge Graph
5. **Формирование промпта** - с учетом эмоций и личности
6. **Генерация ответа** - через LLM провайдер
7. **Верификация** - Truth Loop
8. **Сохранение** - в память
9. **Отправка** - пользователю
10. **Обновление состояния** - через события

Эта архитектура позволяет системе функционировать как автономный когнитивный агент с развитой памятью, эмоциями и способностью к самообучению и адаптации.
