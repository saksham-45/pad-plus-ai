## Начало работы

1. Сделайте форк репозитория
2. Клонируйте ваш форк: `git clone https://github.com/YOUR_USERNAME/padplus-ai.git`
3. Создайте виртуальное окружение и установите зависимости:
   ```bash
   cd padplus-ai
   pip install -r requirements.txt
   cd frontend && npm install && cd ..
   ```
4. Внесите изменения
5. Протестируйте изменения
6. Отправьте pull request

## Структура проекта

```
padplus-ai/
├── backend/                 # Python backend
│   ├── core/               # Основные компоненты системы
│   ├── memory/             # Системы памяти (RAG, факты, эпизодическая и т.д.)
│   ├── emotion/            # Эмоциональные модели (PAD+)
│   ├── knowledge/          # Граф знаний
│   ├── autonomy/           # Автономия и планирование
│   ├── analytics/          # Аналитика и метрики
│   └── llm/                # Провайдеры LLM и интерфейсы
├── frontend/               # React frontend
│   ├── src/                # Исходный код
│   └── public/             # Публичные ресурсы
├── data/                   # Файлы данных (игнорируются git)
├── docs/                   # Документация
├── scripts/                # Скрипты утилит
└── tests/                  # Тесты
```

## Рабочий процесс

1. Создайте ветку для фичи: `git checkout -b feature/my-feature`
2. Внесите изменения
3. Добавьте тесты, если применимо
4. Запустите тесты: `python -m pytest tests/`
5. Следуйте стандартам кодирования (см. ниже)
6. Зафиксируйте изменения с описательным сообщением
7. Загрузите в свой форк
8. Создайте pull request

## Стандарты кода

### Python
- Следуйте руководству по стилю PEP 8
- Используйте аннотации типов для всех параметров функций и возвращаемых значений
- Пишите docstring для всех публичных функций и классов
- Используйте описательные имена переменных и функций
- Держите функции сфокусированными и маленькими, когда это возможно

### JavaScript/React
- Используйте функциональные компоненты с хуками
- Следуйте руководству по стилю JavaScript от Airbnb
- Используйте TypeScript, где это возможно
- Делайте компоненты сфокусированными и переиспользуемыми

### Документация
- Документируйте сложные алгоритмы и архитектурные решения
- Обновляйте README при добавлении новых функций
- Добавляйте комментарии для непонятных участков кода

## Тестирование

Все вклады должны включать соответствующие тесты:

1. Модульные тесты для отдельных функций/классов
2. Интеграционные тесты для взаимодействий системы
3. Сквозные тесты для критических пользовательских сценариев

Запустите все тесты перед отправкой:
```bash
python -m pytest tests/
```

## Pull Request

1. Заполните шаблон pull request
2. Четко опишите свои изменения
3. Укажите связанные задачи
4. Включите скриншоты, если изменяется UI
5. Убедитесь, что все тесты проходят
6. Запросите ревью у мейнтейнеров

## Правила сообщества

### Наше обязательство

В интересах создания открытой и гостеприимной среды мы обязуемся сделать участие в нашем проекте и сообществе опытом без домогательств для всех, независимо от возраста, телосложения, инвалидности, этнической принадлежности, гендерной идентичности и выражения, уровня опыта, национальности, внешности, расы, религии или сексуальной идентичности и ориентации.

### Наши стандарты

Примеры поведения, способствующего созданию благоприятной обстановки, включают:
- Использование доброжелательного и инклюзивного языка
- Уважение к различным точкам зрения и опыту
- Вежливое принятие конструктивной критики
- Сосредоточение внимания на том, что лучше для сообщества
- Проявление эмпатии к другим участникам проекта

Примеры неприемлемого поведения участников включают:
- Использование сексуализированного языка или образов и нежелательное сексуальное внимание или домогательства
- Троллинг, оскорбительные/уничижительные комментарии и личные или политические нападки
- Публичные или частные домогательства
- Публикация чужой частной информации, например, физического или электронного адреса, без явного разрешения
- Другое поведение, которое обоснованно считать неуместным в профессиональной обстановке
