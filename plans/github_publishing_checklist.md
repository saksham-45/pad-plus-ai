# Чеклист подготовки к публикации на GitHub

## Подготовка репозитория

- [ ] Удалить все конфиденциальные данные и API ключи из кода
- [ ] Убедиться, что все чувствительные данные находятся в `.gitignore`
- [ ] Проверить, что `.env` файлы находятся в `.gitignore`
- [ ] Убедиться, что файлы с данными (SQLite базы, JSON файлы состояния) игнорируются
- [ ] Создать `.gitattributes` файл для правильной обработки окончаний строк
- [ ] Проверить, что все зависимости корректно указаны в `requirements.txt`

## Документация

- [ ] Проверить актуальность `README.md` (уже создан в `plans/github_readme.md`)
- [ ] Убедиться, что лицензия корректно указана в `LICENSE` файле
- [ ] Проверить, что `CONTRIBUTING.md` содержит актуальную информацию
- [ ] Убедиться, что `CODE_OF_CONDUCT.md` и `SECURITY.md` актуальны
- [ ] Создать `CHANGELOG.md` с историей изменений
- [ ] Проверить, что документация в `docs/` актуальна

## Код

- [ ] Убедиться, что в коде нет жестко закодированных значений и API ключей
- [ ] Проверить, что все зависимости имеют корректные версии
- [ ] Убедиться, что в коде нет тестовых API ключей или учетных данных
- [ ] Проверить, что все пути к файлам используют относительные ссылки
- [ ] Убедиться, что код соответствует стандартам (PEP 8 для Python)
- [ ] Проверить, что все функции и классы имеют документацию

## Безопасность

- [ ] Проверить, что в репозитории нет файлов с конфиденциальной информацией
- [ ] Убедиться, что все учетные данные находятся в `.env.example`
- [ ] Проверить, что все зависимости обновлены до безопасных версий
- [ ] Убедиться, что в коде реализована защита от основных уязвимостей

## Тестирование

- [ ] Убедиться, что все тесты проходят успешно
- [ ] Проверить, что покрытие тестов удовлетворительное
- [ ] Убедиться, что тесты не зависят от локальной конфигурации
- [ ] Проверить, что тесты могут быть запущены в любой среде

## Структура проекта

- [ ] Убедиться, что структура проекта логична и понятна
- [ ] Проверить, что все важные компоненты находятся в правильных директориях
- [ ] Убедиться, что файлы конфигурации находятся в корне проекта
- [ ] Проверить, что все ресурсы находятся в соответствующих директориях

## CI/CD
