# Настройка OpenRouter на Render.com

## Инструкция для пользователя

### 1. Настройка переменных окружения на Render

1. Зайдите в панель управления Render.com
2. Выберите ваш сервис PAD+ AI
3. Перейдите в раздел **Environment** → **Environment Variables**
4. Добавьте следующие переменные:

```
OPENROUTER_API_KEY=ваш_ключ_от_OpenRouter
OPENROUTER_ENABLED=true
OPENROUTER_MODEL=openrouter/free
```

### 2. Получение OpenRouter API ключа

1. Зарегистрируйтесь на [OpenRouter.ai](https://openrouter.ai)
2. Перейдите в раздел **API Keys**
3. Создайте новый API ключ
4. Скопируйте ключ и вставьте в переменную `OPENROUTER_API_KEY`

### 3. Проверка настроек

После деплоя системы:

1. Откройте Settings в интерфейсе PAD+ AI
2. Выберите вкладку **LLM Providers**
3. Найдите **OpenRouter** в списке
4. Нажмите **Test Connection**

### 4. Ожидаемые результаты

✅ **Статус провайдера**: Включен  
✅ **API ключ**: Найден  
✅ **Тест подключения**: Успешно  
✅ **Модель по умолчанию**: google/gemma-7b-it (бесплатная)

### 5. Доступные бесплатные модели

- `google/gemma-7b-it` (по умолчанию)
- `google/gemma-2-9b-it`
- `meta-llama/llama-3-8b-instruct`

### 6. Решение проблем

#### ❌ "API ключ не настроен"
- Проверьте, что переменная `OPENROUTER_API_KEY` добавлена
- Убедитесь, что ключ скопирован без пробелов

#### ❌ "Ошибка аутентификации"
- Проверьте, что API ключ действителен
- Убедитесь, что `OPENROUTER_ENABLED=true`

#### ❌ "Провайдер отключен"
- Проверьте значение `OPENROUTER_ENABLED=true`
- Перезапустите сервис после изменения переменных

### 7. Дополнительные настройки

#### Использование платных моделей
```
OPENROUTER_MODEL=openai/gpt-4
```

#### Использование других бесплатных моделей
```
OPENROUTER_MODEL=google/gemma-2-9b-it
```

### 8. API эндпоинты

После настройки доступны следующие API:

- `GET /llm/providers` - статус провайдеров
- `GET /llm/models` - список моделей
- `POST /llm/test/openrouter` - тест подключения
- `POST /llm/config` - настройка провайдеров

### 9. Логи

Для диагностики проблем проверьте логи Render:

1. Перейдите в **Logs** в панели Render
2. Ищите сообщения с тегом `OpenRouter`
3. Проверьте наличие ошибок аутентификации

### 10. Поддержка

Если возникнут проблемы:

1. Проверьте эту инструкцию
2. Проверьте логи Render
3. Убедитесь, что API ключ действителен
4. Попробуйте использовать другую модель

---

**Важно**: OpenRouter предоставляет бесплатные модели для тестирования. Для коммерческого использования могут потребоваться платные тарифы.
